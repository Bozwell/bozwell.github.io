git_pull_slack_bot

Terraform code repository를 여러 사람이 함께 사용하다 보면, 코드를 작성할때 다른 사람이 push한것을 pull 하지않아, conflict가 생기는 경우가 종종 있습니다. 어느날 귀찮니즘이 발동하여 이를 자동화할 수 있는 도구를 만들어 보았습니다.

동작 방식은 이렇습니다.

  • 보스웰의 회사는 GitLab, slack를 사용하고 있습니다.
  • GitLab repository에 push 발생시 slack으로 notification 하도록 설정하였습니다.
  • python으로 slackbot를 만들어서 repository이름이 발견되면 local repository를 업데이트 합니다.

사용해보니 많이 편합니다. 하지만 소스코드가 모듈화 되어 있지 않다면 해당 모듈을 코딩하고 있을때 업데이트가 이루어 져서 conflict가 발생할 수 있으므로 주의해야 할것 같
습니다. 보스웰의 팀은 테라폼 코드를 모듈로 세분화해서 이런 경우가 거의 없습니다. Terraform 모듈화 사례에 대해서는 다음 포스트에서 소개해 드리도록 하겠습니다.
